# Phase 0: Discovery & Feasibility

## Purpose

Validate that we understand the problem before designing solutions.

## Activities

| Activity | When to Use | Location |
|----------|-------------|----------|
| Stakeholder Discovery | Conducting interviews, gathering requirements | [stakeholder-discovery-activity/](stakeholder-discovery-activity/SKILL.md) |
| Feasibility Assessment | Evaluating technical/business/resource viability | [feasibility-assessment-activity/](feasibility-assessment-activity/SKILL.md) |
| Discovery Synthesis | Producing validated Phase 0 artifacts | [discovery-synthesis-activity/](discovery-synthesis-activity/SKILL.md) |

## Phase Flow
┌─────────────────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│    Stakeholder      │     │    Feasibility      │     │     Discovery       │
│     Discovery       │────▶│    Assessment       │────▶│     Synthesis       │
│   (interviews)      │     │    (analysis)       │     │    (artifacts)      │
└─────────────────────┘     └─────────────────────┘     └─────────────────────┘

## Entry Criteria

- Project or initiative identified
- Executive sponsor available
- Stakeholders identified for interviews

## Exit Criteria

- [ ] Problem statement validated by stakeholders
- [ ] Feasibility confirmed (or no-go decision made)
- [ ] Risks documented and assessed
- [ ] Success criteria defined
- [ ] Stakeholder alignment achieved
